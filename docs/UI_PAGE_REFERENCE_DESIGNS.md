### Key Features

- **Widget URL:** Copyable URL for embedding
- **API Key:** Masked with show/copy/rotate actions
- **Security Toggle:** Checkbox for key requirement
- **Clear Sections:** Organized by function

## Design Principles Applied

### 1. Visual Hierarchy
- **Clear Headings:** Section titles with dividers
- **Grouped Content:** Related settings together
- **Whitespace:** Generous spacing between sections

### 2. Navigation
- **Sticky Sidebar:** Always visible, shows active state
- **Breadcrumbs:** (Optional) Show current location
- **User Menu:** Top-right dropdown for account access

### 3. Forms & Inputs
- **Clear Labels:** Descriptive field labels
- **Help Text:** Explanations below inputs
- **Validation:** Real-time feedback
- **Save Actions:** Per-section or global save

### 4. Actions
- **Primary Actions:** Prominent buttons (Save, Invite)
- **Destructive Actions:** Red/destructive styling (Delete)
- **Secondary Actions:** Outlined buttons (Cancel, Reset)

### 5. Feedback
- **Loading States:** Spinners during async operations
- **Success Messages:** Toast notifications
- **Error Messages:** Inline or toast errors

---

## Color & Typography

### Colors (Tailwind-based)
- **Primary:** Indigo-600 (buttons, links)
- **Destructive:** Red-600 (delete actions)
- **Text:** Gray-900 (headings), Gray-700 (body)
- **Borders:** Gray-200 (dividers, inputs)
- **Background:** White (cards), Gray-50 (page)

### Typography
- **Headings:** Bold, larger sizes (text-2xl, text-xl)
- **Body:** Regular weight, readable size (text-sm, text-base)
- **Labels:** Medium weight, small size (text-sm font-medium)
- **Help Text:** Light gray, small size (text-xs text-gray-500)

---

## Responsive Design

### Mobile (< 768px)
- **Sidebar:** Collapsible/hamburger menu
- **Tables:** Stack vertically or scroll horizontally
- **Forms:** Full-width inputs
- **Buttons:** Full-width on mobile

### Tablet (768px - 1024px)
- **Sidebar:** Collapsible, can be toggled
- **Content:** Adjusted spacing
- **Tables:** Horizontal scroll if needed

### Desktop (> 1024px)
- **Sidebar:** Always visible, fixed width
- **Content:** Max-width container, centered
- **Tables:** Full table display

## Implementation Notes

### Page Structure Pattern

```tsx
// Example: /admin/account/profile/page.tsx
export default function ProfilePage() {
  return (
    <SettingsLayout
      sidebarItems={[
        { href: '/admin/account/profile', label: 'Profile', active: true },
        { href: '/admin/account/security', label: 'Security' },
        { href: '/admin/account/preferences', label: 'Preferences' },
      ]}
    >
      <SectionCard title="Profile" description="Manage your profile information">
        {/* Profile form content */}
      </SectionCard>
    </SettingsLayout>
  );
}
```

### Key Patterns
- **Consistent Layout:** Same structure across all settings pages
- **Reusable Components:** Shared sidebar, cards, forms
- **Type Safety:** TypeScript for all components
- **Accessibility:** ARIA labels, keyboard navigation
